---
name: implementation-plan
description: Create and maintain an implementation plan for non-trivial work with checkable steps and explicit verification.
---

Use this skill when:

- A task has multiple steps
- Requirements are ambiguous
- The work spans architecture, integration, or multiple files
- A progress tracker would help over multiple turns

Workflow:

1. Restate the goal in 1-3 sentences.
2. Identify assumptions, risks, and unknowns.
3. Write a short checklist plan in `tasks/todo.md`.
4. Include verification steps in the checklist, not only build steps.
5. Update checklist status as work progresses.
6. Add a short review/results section at the end of `tasks/todo.md`.
