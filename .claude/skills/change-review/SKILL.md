---
name: change-review
description: Review a proposed solution for elegance, correctness, and unnecessary complexity before presenting it.
---

Use this skill when:

- A change is non-trivial
- The fix may be overly complex
- The solution should be challenged before finalizing

Workflow:

1. Check whether the solution is simpler than necessary.
2. Check whether the fix addresses the root cause.
3. Check whether unrelated files or concerns were touched.
4. Check whether verification matches the risk of the change.
5. Produce a short review note with any recommended simplifications.
