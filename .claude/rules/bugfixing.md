---
description: Expectations when handling bug reports
---

- Start by identifying the root cause using logs, errors, failing tests, or reproduction steps.
- Fix the actual cause, not just the visible symptom, unless a temporary mitigation is explicitly requested.
- Reduce user hand-holding; drive the investigation forward independently when enough evidence exists.
- Add or update verification so the bug is less likely to recur.
- If the fix is a workaround rather than a true solution, clearly document the underlying issue and next steps for a proper fix.
